Introduction à l'authentification LEXA API

LEXA API utilise l'authentification par Bearer Token (clé API) pour sécuriser l'accès aux endpoints. Cette méthode simple et sécurisée permet d'identifier votre compte et de valider vos permissions.

Avantages de l'authentification par clé API

  • Simple : Un seul header HTTP à ajouter
  • Sécurisé : Communication HTTPS chiffrée
  • Révocable : Régénérez votre clé à tout moment
  • Sans session : Stateless, pas de cookies ni de sessions
  • Traçable : Toutes les requêtes sont liées à votre compte

Obtenir votre clé API

Prérequis

  • Un compte LEXA actif
  • Un abonnement LEXA API souscrit

Étapes pour obtenir votre clé

  1. Connectez-vous à portail.lexamt.fr
  2. Accédez à Paramètres dans le menu principal
  3. Cliquez sur API ou Clé API dans le sous-menu
  4. Votre clé API s'affiche (si vous en avez déjà une)
  5. Si vous n'avez pas de clé, cliquez sur Générer une clé API
  6. La clé s'affiche : cliquez sur Copier pour la copier dans le presse-papier
  7. Conservez cette clé en lieu sûr (vous ne pourrez plus la visualiser ultérieurement)

Format de la clé API

Une clé API LEXA est une chaîne de caractères alphanumériques d'environ 36 caractères.

Exemple : abc123def456ghi789jkl012mno345pq

Important : Cette clé est unique et confidentielle. Ne la partagez jamais publiquement.

Utiliser votre clé API

Format du header Authorization

Toutes les requêtes à LEXA API doivent inclure le header HTTP suivant :

Authorization: Bearer VOTRE_CLE_API

Exemples par méthode

Avec curl

curl -X GET https://portail.lexamt.fr/api/v1/languages/ \
  -H "Authorization: Bearer abc123def456ghi789jkl012mno345pq"

Avec Python (requests)

import requests

API_KEY = "abc123def456ghi789jkl012mno345pq"

headers = {
    "Authorization": f"Bearer {API_KEY}"
}

response = requests.get(
    "https://portail.lexamt.fr/api/v1/languages/",
    headers=headers
)

print(response.json())

Avec JavaScript (fetch)

const API_KEY = 'abc123def456ghi789jkl012mno345pq';

const headers = {
    'Authorization': `Bearer ${API_KEY}`
};

fetch('https://portail.lexamt.fr/api/v1/languages/', {
    headers: headers
})
.then(response => response.json())
.then(data => console.log(data));

Avec PHP

<?php
$apiKey = 'abc123def456ghi789jkl012mno345pq';

$options = array(
    'http' => array(
        'header' => "Authorization: Bearer $apiKey\r\n"
    )
);

$context = stream_context_create($options);
$result = file_get_contents(
    'https://portail.lexamt.fr/api/v1/languages/',
    false,
    $context
);

$data = json_decode($result, true);
print_r($data);
?>

Processus de validation

Comment LEXA API valide votre clé

Lorsque vous envoyez une requête, LEXA API effectue les vérifications suivantes :

  1. Présence du header : Le header Authorization doit être présent
  2. Format du header : Doit commencer par "Bearer " suivi de la clé
  3. Validité de la clé : La clé doit correspondre à un compte actif
  4. Abonnement actif : Le compte doit avoir un abonnement LEXA API actif
  5. Type d'abonnement : L'abonnement doit autoriser l'usage de l'API

Traitement de la requête

Erreurs d'authentification

Header Authorization manquant

Code HTTP : 401 Unauthorized

{
  "detail": "Header Authorization requis"
}

Cause : Vous n'avez pas inclus le header Authorization dans votre requête.

Solution : Ajoutez le header Authorization: Bearer VOTRE_CLE

Format du header incorrect

Code HTTP : 400 Bad Request

{
  "detail": "Le header Authorization doit avoir le format: Bearer "
}

Causes possibles :

  • Vous avez oublié le mot "Bearer" avant la clé
  • Il manque un espace entre "Bearer" et la clé
  • Vous avez utilisé un autre type d'authentification (Basic, etc.)

Solution : Format correct : Authorization: Bearer abc123...

Clé API manquante après "Bearer"

Code HTTP : 400 Bad Request

{
  "detail": "Clé API requise après 'Bearer'"
}

Cause : Le header contient "Bearer " mais la clé est vide.

Solution : Vérifiez que vous avez bien collé votre clé API après "Bearer ".

Clé API trop longue

Code HTTP : 400 Bad Request

{
  "detail": "La clé API ne peut pas dépasser 36 caractères"
}

Cause : La clé fournie dépasse 36 caractères.

Solution : Vérifiez que vous n'avez pas copié d'espaces ou de caractères superflus.

Clé API invalide

Code HTTP : 401 Unauthorized

{
  "detail": "Clé API invalide"
}

Causes possibles :

  • La clé ne correspond à aucun compte
  • La clé a été révoquée
  • La clé a été mal copiée (caractères manquants ou en trop)

Solution : Vérifiez votre clé dans le portail LEXA ou générez-en une nouvelle.

Pas d'abonnement actif

Code HTTP : 401 Unauthorized

{
  "detail": "Aucun abonnement actif trouvé pour cette clé API"
}

Cause : Votre abonnement LEXA API est expiré ou inactif.

Solution : Renouvelez votre abonnement sur portail.lexamt.fr.

Plusieurs abonnements actifs

Code HTTP : 400 Bad Request

{
  "detail": "Plusieurs abonnements actifs trouvés. Contactez le support."
}

Cause : Configuration anormale du compte (plusieurs abonnements API actifs).

Solution : Contactez le support LEXA à support@lexamt.fr.

Sécurité de la clé API

Bonnes pratiques de sécurité

1. Ne jamais exposer la clé publiquement

  • Ne commitez jamais la clé dans Git, GitHub, GitLab, etc.
  • Ne la partagez pas sur des forums, Stack Overflow, etc.
  • Ne l'incluez pas dans du code client (JavaScript navigateur)
  • Ne l'envoyez pas par email non chiffré

2. Utiliser des variables d'environnement

Exemple .env :

LEXA_API_KEY=abc123def456ghi789jkl012mno345pq

Exemple Python :

import os
from dotenv import load_dotenv

load_dotenv()
API_KEY = os.getenv('LEXA_API_KEY')

Exemple Node.js :

require('dotenv').config();
const API_KEY = process.env.LEXA_API_KEY;

3. Limiter l'accès à la clé

  • Stockez la clé dans un gestionnaire de secrets sécurisé
  • Limitez qui peut voir la clé dans votre équipe
  • Utilisez des permissions de fichiers restrictives (chmod 600)

4. Utiliser HTTPS uniquement

LEXA API force HTTPS pour toutes les connexions. Ne tentez jamais d'utiliser HTTP, car :

  • Votre clé serait transmise en clair
  • Les requêtes HTTP seront rejetées

5. Rotation régulière

  • Changez votre clé API périodiquement (tous les 3-6 mois)
  • Régénérez immédiatement si vous suspectez une compromission
  • Gardez une trace de quand la clé a été générée

Que faire si votre clé est compromise ?

  1. Révoquez immédiatement : Connectez-vous au portail LEXA
  2. Générez une nouvelle clé : Dans Paramètres > API
  3. Mettez à jour votre code : Remplacez l'ancienne clé partout
  4. Vérifiez l'usage : Consultez l'historique d'utilisation pour détecter des abus
  5. Contactez le support : Signalez l'incident si nécessaire

Régénérer votre clé API

Quand régénérer ?

  • Votre clé a été exposée publiquement (commit Git, forum, etc.)
  • Un employé qui avait accès à la clé a quitté l'entreprise
  • Vous détectez une utilisation suspecte de l'API
  • Rotation de sécurité régulière (recommandé tous les 3-6 mois)
  • Changement d'infrastructure ou de fournisseur

Procédure de régénération

  1. Connectez-vous à portail.lexamt.fr
  2. Accédez à Paramètres > API
  3. Cliquez sur Régénérer la clé API
  4. Confirmez l'action (l'ancienne clé cessera de fonctionner immédiatement)
  5. Copiez la nouvelle clé affichée
  6. Mettez à jour votre code et vos configurations avec la nouvelle clé
  7. Testez que la nouvelle clé fonctionne correctement

⚠️ Attention : La régénération révoque immédiatement l'ancienne clé. Assurez-vous de mettre à jour tous vos services avant de régénérer.

Stratégie de migration sans interruption

Pour éviter les interruptions de service lors de la rotation de clé :

  1. Préparez votre code pour accepter une nouvelle clé (via variable d'environnement)
  2. Régénérez la clé dans le portail
  3. Mettez à jour rapidement la variable d'environnement
  4. Redémarrez vos services un par un
  5. Vérifiez que tous les services fonctionnent avec la nouvelle clé

Surveillance et audit

Surveiller l'utilisation de votre clé

Dans le portail LEXA, vous pouvez :

  • Voir l'historique d'utilisation de votre API
  • Consulter le nombre de requêtes effectuées
  • Vérifier votre quota d'utilisation restant
  • Détecter des pics d'utilisation inhabituels

Logs d'authentification

Pour vos propres logs d'application, enregistrez :

  • Les tentatives d'authentification réussies et échouées
  • Les codes d'erreur retournés par l'API
  • Les timestamps de chaque requête
  • Les endpoints appelés

Important : Ne loguez jamais la clé API complète. Si nécessaire, loguez seulement les 4 derniers caractères.

Limitations de l'authentification

Une clé par compte

Chaque compte LEXA dispose d'une seule clé API active à la fois. Si vous régénérez une clé, l'ancienne est immédiatement révoquée.

Pas d'authentification multi-utilisateur

La clé API est liée à votre compte entier, pas à un utilisateur individuel. Toutes les requêtes avec cette clé sont attribuées au même compte.

Pas de scopes ou permissions

Actuellement, une clé API donne accès à tous les endpoints LEXA API. Il n'est pas possible de créer des clés avec des permissions limitées.

Rate limiting

Même avec une clé valide, votre compte peut être soumis à des limites de taux (rate limiting) selon votre abonnement. Voir la documentation sur les limites.

Dépannage

Ma clé fonctionne par moments, pas toujours

Causes possibles :

  • Problème de connexion Internet intermittent
  • Vous atteignez les limites de taux (rate limiting)
  • Votre abonnement a expiré entre-temps
  • Problème de cache côté client (ancienne clé en cache)

Solutions :

  • Vérifiez votre connexion
  • Consultez votre quota dans le portail LEXA
  • Vérifiez que votre abonnement est actif
  • Videz les caches et redémarrez vos services

J'obtiens une erreur "CORS" avec ma clé

Cause : Vous essayez d'appeler l'API depuis JavaScript navigateur (frontend).

Solution : L'API doit être appelée depuis votre backend (serveur), jamais depuis le frontend. Créez un endpoint sur votre serveur qui appelle LEXA API avec la clé sécurisée.

La clé semble valide mais je reçois des erreurs 401

Vérifications :

  1. Vérifiez qu'il n'y a pas d'espaces avant/après la clé
  2. Vérifiez le format exact : Authorization: Bearer VOTRE_CLE
  3. Assurez-vous que "Bearer" a un "B" majuscule
  4. Testez avec curl pour isoler le problème
  5. Régénérez une nouvelle clé pour être sûr

FAQ sur l'authentification

Puis-je avoir plusieurs clés API ?

Non, chaque compte ne peut avoir qu'une seule clé API active à la fois.

Puis-je partager ma clé avec un collègue ?

Ce n'est pas recommandé pour des raisons de sécurité et de traçabilité. Chaque personne devrait avoir son propre compte LEXA.

Ma clé expire-t-elle ?

Non, les clés API LEXA n'expirent pas automatiquement. Elles restent valides tant que votre abonnement est actif. Il est cependant recommandé de les régénérer régulièrement pour des raisons de sécurité.

Que se passe-t-il si mon abonnement expire ?

Si votre abonnement expire, votre clé API deviendra invalide et vous recevrez des erreurs 401 "Aucun abonnement actif". Renouvelez votre abonnement pour réactiver la clé.

Puis-je utiliser la même clé en développement et en production ?

Techniquement oui, mais ce n'est pas recommandé. Idéalement, utilisez des comptes LEXA distincts pour le développement et la production afin d'isoler les environnements et faciliter le suivi.
