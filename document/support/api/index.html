Introduction à LEXA API

Qu'est-ce que LEXA API ?

LEXA API est une interface de programmation REST qui permet d'intégrer les services de traduction juridique LEXA dans vos applications, sites web et workflows. Elle offre des traductions de haute qualité spécialisées dans le domaine juridique.

Fonctionnalités principales

  • Traduction de texte : Traduisez du texte brut jusqu'à 2000 caractères
  • Traduction de documents : Supportez TXT, DOCX, XLSX, PPTX et PDF (jusqu'à 50MB par fichier)
  • Gestion de glossaires : Créez, consultez et supprimez vos glossaires personnalisés
  • Domaines de spécialisation : Accédez à des traductions spécialisées par domaine juridique
  • Langues multiples : Supportez de nombreuses paires de langues
  • Batch processing : Traduisez jusqu'à 10 fichiers simultanément

Avantages de LEXA API

  • Qualité juridique : Traductions spécialisées avec terminologie juridique précise
  • RESTful : Architecture simple et intuitive basée sur HTTP
  • Sécurisé : Authentification par clé API et communication HTTPS
  • Fiable : API stable avec gestion d'erreurs complète
  • Flexible : Support JSON et multipart/form-data
  • Documenté : Documentation complète avec exemples de code

Cas d'usage

  • Automatisation de traduction de contrats et documents juridiques
  • Intégration dans des systèmes de gestion documentaire (DMS)
  • Création d'interfaces utilisateur personnalisées de traduction
  • Workflows de traduction en masse
  • Applications web et mobiles nécessitant de la traduction juridique
  • Intégration dans des outils métier (CRM, ERP, etc.)

Démarrage rapide

Étape 1 : Obtenir une clé API

  1. Créez un compte sur portail.lexamt.fr
  2. Souscrivez à un abonnement LEXA API (voir section Forfait LEXA API)
  3. Dans votre compte, accédez à Paramètres > API
  4. Générez ou copiez votre clé API
  5. Conservez cette clé en sécurité (ne jamais la partager publiquement)

Étape 2 : Première requête API

Voici un exemple simple de traduction de texte avec curl :

curl -X POST https://portail.lexamt.fr/api/v1/translate/ \
  -H "Authorization: Bearer VOTRE_CLE_API" \
  -H "Content-Type: application/json" \
  -d '{
    "action": "text_translate",
    "source_language": "en",
    "target_language": "fr",
    "text": "This is a legal contract."
  }'

Étape 3 : Traiter la réponse

Réponse attendue :

{
  "translated_text": "Il s'agit d'un contrat juridique."
}

Pour plus d'exemples détaillés, consultez le Guide de démarrage rapide et la page Exemples de code.

Authentification

Méthode d'authentification

LEXA API utilise l'authentification par Bearer Token (clé API).

Format du header

Authorization: Bearer VOTRE_CLE_API

Exemple complet

GET /api/v1/languages/ HTTP/1.1
Host: portail.lexamt.fr
Authorization: Bearer abc123def456ghi789jkl012mno345pq
Content-Type: application/json

Obtenir votre clé API

  1. Connectez-vous à portail.lexamt.fr
  2. Accédez à Paramètres dans votre compte
  3. Cliquez sur API ou Clé API
  4. Cliquez sur Générer une clé ou copiez la clé existante

Sécurité de la clé API

  • Ne jamais partager votre clé API publiquement (GitHub, forums, etc.)
  • Ne jamais exposer la clé dans le code frontend (JavaScript client)
  • Utiliser des variables d'environnement pour stocker la clé
  • Régénérer la clé si elle est compromise
  • Limiter les permissions si possible (selon votre abonnement)

Révoquer ou régénérer une clé

Pour révoquer votre clé actuelle et en générer une nouvelle :

  1. Connectez-vous au portail LEXA
  2. Accédez à Paramètres > API
  3. Cliquez sur Régénérer la clé
  4. Confirmez l'action (l'ancienne clé cessera immédiatement de fonctionner)
  5. Mettez à jour votre code avec la nouvelle clé

Pour plus de détails, consultez le Guide d'authentification complet.

Endpoints de l'API

LEXA API expose plusieurs endpoints organisés par fonctionnalité. Tous les endpoints sont préfixés par /api/v1/.

Vue d'ensemble des endpoints

Endpoint Méthode Description
Traduction
/api/v1/translate/ POST Traduire du texte ou des fichiers
Langues
/api/v1/languages/ GET Liste des langues disponibles
Domaines
/api/v1/domains/ GET Liste des domaines de spécialisation
/api/v1/domain/{id}/glossaries/ GET Glossaires par défaut d'un domaine
Glossaires
/api/v1/glossaries/ GET Liste vos glossaires personnels
/api/v1/glossary/ POST Créer un nouveau glossaire
/api/v1/glossary/{id}/ GET Récupérer un glossaire spécifique
/api/v1/glossary/{id}/ DELETE Supprimer un glossaire

Endpoint de traduction

POST /api/v1/translate/

Cet endpoint permet deux types de traduction :

1. Traduction de texte
{
  "action": "text_translate",
  "source_language": "en",
  "target_language": "fr",
  "text": "Your text here",
  "domain_name": "Contract Law" (optionnel)
}
2. Traduction de fichiers
{
  "action": "file_translate",
  "source_language": "en",
  "target_language": "fr",
  "document": ["base64_encoded_file_1", "base64_encoded_file_2"],
  "domain_name": "Contract Law" (optionnel)
}

Formats supportés : TXT, DOCX, XLSX, PPTX, PDF

Taille maximale : 50MB par fichier, 10 fichiers maximum par requête

Endpoint de langues

GET /api/v1/languages/

Retourne la liste des langues disponibles pour la traduction.

Réponse :

[
  {
    "name": "English",
    "language_code": "en"
  },
  {
    "name": "French",
    "language_code": "fr"
  }
]

Endpoint de domaines

GET /api/v1/domains/

Retourne la liste des domaines de spécialisation juridique.

Réponse :

[
  {
    "id": 1,
    "domain_group": "Legal",
    "name": "Contract Law"
  },
  {
    "id": 2,
    "domain_group": "Legal",
    "name": "Intellectual Property"
  }
]

Endpoints de glossaires

GET /api/v1/glossaries/

Liste tous vos glossaires personnels.

POST /api/v1/glossary/

Crée un nouveau glossaire.

{
  "name": "Mon glossaire juridique",
  "source_language": "en",
  "target_language": "fr",
  "file": "base64_encoded_csv_or_xlsx"
}

Formats acceptés : CSV, XLSX

Taille maximale : 10MB

GET /api/v1/glossary/{id}/

Récupère les détails d'un glossaire spécifique par son ID.

DELETE /api/v1/glossary/{id}/

Supprime un glossaire par son ID.

Pour la documentation complète avec tous les paramètres et exemples, consultez la Référence des endpoints.

Gestion des erreurs

Format des erreurs

Toutes les erreurs sont retournées au format JSON suivant :

{
  "detail": "Message d'erreur descriptif"
}

Codes HTTP

Code Signification Description
200 OK Requête réussie avec contenu
204 No Content Requête réussie sans contenu (ex: suppression)
400 Bad Request Erreur de validation, données manquantes ou invalides
401 Unauthorized Authentification manquante ou invalide
404 Not Found Ressource non trouvée
500 Internal Server Error Erreur serveur interne

Erreurs courantes

Erreur d'authentification (401)

{
  "detail": "Header Authorization requis"
}

Cause : Header Authorization manquant ou clé API invalide

Solution : Vérifiez que le header Authorization: Bearer VOTRE_CLE est présent

Erreur de validation (400)

{
  "detail": "Le champ 'text' est requis pour l'action 'text_translate'"
}

Cause : Champ requis manquant ou invalide

Solution : Vérifiez que tous les champs requis sont présents et valides

Ressource non trouvée (404)

{
  "detail": "Glossaire non trouvé"
}

Cause : L'ID du glossaire n'existe pas ou n'appartient pas à votre compte

Solution : Vérifiez l'ID et vos permissions

Pour la liste complète des codes d'erreur, consultez la page Codes d'erreur.

Limites et quotas

Limites techniques

Élément Limite
Longueur du texte (traduction) 2000 caractères maximum
Taille d'un fichier (traduction) 50MB maximum
Nombre de fichiers par requête 10 fichiers maximum
Taille d'un fichier glossaire 10MB maximum
Longueur nom de glossaire 128 caractères maximum
Longueur code de langue 10 caractères maximum
Longueur nom de domaine 128 caractères maximum
Longueur clé API 36 caractères maximum

Formats de fichiers supportés

Traduction de documents

  • TXT (texte brut)
  • DOCX (Microsoft Word)
  • XLSX (Microsoft Excel)
  • PPTX (Microsoft PowerPoint)
  • PDF (Portable Document Format)

Glossaires

  • CSV (Comma-Separated Values)
  • XLSX (Microsoft Excel)

Quotas d'utilisation

Les quotas d'utilisation dépendent de votre abonnement LEXA API. Consultez la section Forfait LEXA API pour plus de détails.

Rate limiting

Pour garantir la qualité de service pour tous les utilisateurs, LEXA API peut appliquer des limites de taux (rate limiting). Si vous dépassez ces limites, vous recevrez une erreur HTTP 429 (Too Many Requests).

Bonnes pratiques :

  • Espacez vos requêtes au lieu de les envoyer toutes simultanément
  • Implémentez un système de retry avec backoff exponentiel
  • Utilisez le batch processing (10 fichiers à la fois) pour optimiser
  • Contactez le support pour des besoins de volume élevé

Communauté

Bonnes pratiques

Sécurité

  • Stockez votre clé API dans des variables d'environnement, pas dans le code
  • Ne jamais exposer la clé API côté client (JavaScript navigateur)
  • Utilisez HTTPS pour toutes vos requêtes
  • Régénérez votre clé si elle est compromise
  • Limitez les permissions de la clé si possible

Performance

  • Utilisez le batch processing pour traduire plusieurs fichiers
  • Cachez les résultats des endpoints GET (langues, domaines)
  • Implémentez un système de retry avec backoff exponentiel
  • Compressez les fichiers avant envoi si possible
  • Utilisez des requêtes asynchrones pour ne pas bloquer votre application

Gestion des erreurs

  • Vérifiez toujours le code HTTP de la réponse
  • Loguez les erreurs pour faciliter le débogage
  • Affichez des messages d'erreur conviviaux aux utilisateurs
  • Implémentez des fallbacks pour les erreurs réseau
  • Testez votre gestion d'erreurs en environnement de développement

Optimisation des coûts

  • Évitez les traductions redondantes (utilisez un cache)
  • Traduisez uniquement ce qui est nécessaire
  • Regroupez les traductions pour optimiser les appels API
  • Surveillez votre utilisation dans le portail LEXA
