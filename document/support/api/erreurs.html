Introduction

Cette page documente tous les codes d'erreur possibles de LEXA API, leurs causes et les solutions recommandées.

Format des erreurs

Toutes les erreurs API sont retournées au format JSON suivant :

{
  "detail": "Message d'erreur descriptif"
}

Le code HTTP accompagnant la réponse indique le type d'erreur.

Codes HTTP

Code Nom Description Quand survient-il ?
200 OK Requête réussie avec contenu Succès de la traduction, récupération de données
204 No Content Requête réussie sans contenu de retour Suppression réussie d'un glossaire
400 Bad Request Erreur de validation, données invalides Champ manquant, format incorrect, limite dépassée
401 Unauthorized Authentification manquante ou échouée Clé API manquante, invalide, ou abonnement inactif
404 Not Found Ressource non trouvée Glossaire inexistant, endpoint invalide
429 Too Many Requests Limite de taux dépassée Trop de requêtes dans un court laps de temps
500 Internal Server Error Erreur interne du serveur Problème temporaire côté serveur

Erreurs d'authentification

Header Authorization manquant

Code HTTP : 401

{
  "detail": "Header Authorization requis"
}

Cause : Le header Authorization n'est pas présent dans la requête.

Solution : Ajoutez le header Authorization: Bearer VOTRE_CLE_API à toutes vos requêtes.

Format du header Authorization invalide

Code HTTP : 400

{
  "detail": "Le header Authorization doit avoir le format: Bearer <API_KEY>"
}

Causes possibles :

  • Le mot "Bearer" est manquant
  • Il manque un espace entre "Bearer" et la clé
  • Utilisation d'un autre type d'auth (Basic, etc.)

Solution : Format correct : Authorization: Bearer abc123...

Clé API manquante après "Bearer"

Code HTTP : 400

{
  "detail": "Clé API requise après 'Bearer'"
}

Cause : Le header contient "Bearer " mais la clé est vide.

Solution : Assurez-vous que votre clé API est bien présente après "Bearer ".

Clé API trop longue

Code HTTP : 400

{
  "detail": "La clé API ne peut pas dépasser 36 caractères"
}

Cause : La clé fournie dépasse 36 caractères.

Solution : Vérifiez que vous n'avez pas copié d'espaces ou caractères superflus avec la clé.

Clé API invalide

Code HTTP : 401

{
  "detail": "Clé API invalide"
}

Causes possibles :

  • La clé ne correspond à aucun compte
  • La clé a été révoquée ou régénérée
  • La clé est mal copiée (caractères manquants ou en trop)

Solution : Vérifiez votre clé dans le portail LEXA ou générez-en une nouvelle.

Aucun abonnement actif

Code HTTP : 401

{
  "detail": "Aucun abonnement actif trouvé pour cette clé API"
}

Cause : Votre abonnement LEXA API est expiré ou inactif.

Solution : Renouvelez votre abonnement sur portail.lexamt.fr.

Plusieurs abonnements actifs détectés

Code HTTP : 400

{
  "detail": "Plusieurs abonnements actifs trouvés. Veuillez contacter le support."
}

Cause : Configuration anormale du compte (plusieurs abonnements API actifs simultanément).

Solution : Contactez le support LEXA : support@lexamt.fr

Erreurs de validation

Champ requis manquant

Code HTTP : 400

{
  "detail": "Le champ 'action' est requis"
}

{
  "detail": "Le champ 'source_language' est requis"
}

{
  "detail": "Le champ 'target_language' est requis"
}

{
  "detail": "Le champ 'text' est requis pour l'action 'text_translate'"
}

{
  "detail": "Le champ 'document' est requis pour l'action 'file_translate'"
}

Cause : Un champ obligatoire n'est pas fourni dans la requête.

Solution : Incluez tous les champs requis selon l'action :

  • text_translate : action, source_language, target_language, text
  • file_translate : action, source_language, target_language, document

Action invalide

Code HTTP : 400

{
  "detail": "Action invalide. Valeurs acceptées: text_translate, file_translate"
}

Cause : Le champ "action" ne contient pas une valeur valide.

Solution : Utilisez uniquement "text_translate" ou "file_translate".

Texte trop long

Code HTTP : 400

{
  "detail": "Le texte ne peut pas dépasser 2000 caractères"
}

Cause : Le texte à traduire dépasse 2000 caractères.

Solution : Divisez le texte en segments plus courts ou utilisez la traduction de fichiers.

Code de langue invalide

Code HTTP : 400

{
  "detail": "Code de langue invalide ou trop long (maximum 10 caractères)"
}

Cause : Le code de langue est trop long ou dans un format incorrect.

Solution : Utilisez des codes ISO standard (ex: "en", "fr", "es") de 10 caractères maximum.

Langues source et cible identiques

Code HTTP : 400

{
  "detail": "Les langues source et cible doivent être différentes"
}

Cause : Vous essayez de traduire d'une langue vers la même langue.

Solution : Assurez-vous que source_language et target_language sont différents.

Nom de domaine trop long

Code HTTP : 400

{
  "detail": "Le nom du domaine ne peut pas dépasser 128 caractères"
}

Cause : Le nom du domaine fourni est trop long.

Solution : Utilisez un nom de domaine de 128 caractères maximum.

Nom de glossaire trop long

Code HTTP : 400

{
  "detail": "Le nom du glossaire ne peut pas dépasser 128 caractères"
}

Cause : Le nom du glossaire dépasse 128 caractères.

Solution : Choisissez un nom plus court (max 128 caractères).

Trop de fichiers

Code HTTP : 400

{
  "detail": "Maximum 10 fichiers par requête"
}

Cause : Vous essayez de traduire plus de 10 fichiers simultanément.

Solution : Divisez votre batch en groupes de 10 fichiers maximum.

Erreurs de fichiers

Fichier base64 invalide

Code HTTP : 400

{
  "detail": "Le fichier n'est pas un base64 valide"
}

Cause : Le contenu fourni n'est pas un encodage base64 valide.

Solution : Vérifiez que vous encodez correctement le fichier en base64 avant l'envoi.

Type de fichier non supporté

Code HTTP : 400

{
  "detail": "Type de fichier non supporté. Formats acceptés: TXT, DOCX, XLSX, PPTX, PDF"
}

{
  "detail": "Type de glossaire non supporté. Formats acceptés: CSV, XLSX"
}

Cause : Le format du fichier n'est pas supporté.

Solution :

  • Traduction : Utilisez TXT, DOCX, XLSX, PPTX ou PDF
  • Glossaires : Utilisez CSV ou XLSX

Fichier trop volumineux

Code HTTP : 400

{
  "detail": "Le fichier ne peut pas dépasser 50MB"
}

{
  "detail": "Le fichier glossaire ne peut pas dépasser 10MB"
}

Cause : Le fichier dépasse la taille maximale autorisée.

Solution :

  • Compressez le fichier si possible
  • Divisez les gros documents en plusieurs fichiers
  • Traduction : Max 50MB par fichier
  • Glossaires : Max 10MB

Fichier vide

Code HTTP : 400

{
  "detail": "Le fichier est vide"
}

Cause : Le fichier uploadé ne contient aucune donnée.

Solution : Vérifiez que le fichier contient bien du contenu avant de l'envoyer.

Signature de fichier invalide

Code HTTP : 400

{
  "detail": "La signature du fichier ne correspond pas à l'extension déclarée"
}

Cause : Le contenu du fichier ne correspond pas à son extension (ex: un fichier .txt qui est en réalité un PDF).

Solution : Assurez-vous que l'extension du fichier correspond à son vrai format.

Erreurs de ressources

Langue inexistante

Code HTTP : 400

{
  "detail": "Langue source inexistante"
}

{
  "detail": "Langue cible inexistante"
}

Cause : Le code de langue fourni ne correspond à aucune langue disponible.

Solution : Utilisez GET /api/v1/languages/ pour obtenir la liste des langues valides.

Glossaire non trouvé

Code HTTP : 404

{
  "detail": "Glossaire non trouvé"
}

Causes possibles :

  • L'ID du glossaire n'existe pas
  • Le glossaire appartient à un autre utilisateur
  • Le glossaire a déjà été supprimé

Solution : Vérifiez l'ID du glossaire et vos permissions.

ID de ressource invalide

Code HTTP : 400

{
  "detail": "ID invalide (doit être un entier PostgreSQL valide: 0-2147483647)"
}

Cause : L'ID fourni n'est pas un entier valide ou dépasse les limites PostgreSQL.

Solution : Utilisez un entier entre 0 et 2147483647.

Domaine non trouvé

Code HTTP : 404

{
  "detail": "Domaine non trouvé"
}

Cause : Le domaine spécifié n'existe pas.

Solution : Utilisez GET /api/v1/domains/ pour obtenir la liste des domaines valides.

Erreurs serveur

Erreur interne du serveur

Code HTTP : 500

{
  "detail": "Une erreur interne s'est produite. Veuillez réessayer."
}

Cause : Problème temporaire côté serveur.

Solutions :

  • Réessayez après quelques secondes
  • Implémentez un retry avec backoff exponentiel
  • Si le problème persiste, contactez le support

Service temporairement indisponible

Code HTTP : 503

{
  "detail": "Service temporairement indisponible. Veuillez réessayer plus tard."
}

Cause : Maintenance ou surcharge du serveur.

Solution : Attendez quelques minutes et réessayez.

Timeout de requête

Code HTTP : 504

{
  "detail": "La requête a expiré (timeout)"
}

Causes possibles :

  • Fichier très volumineux à traiter
  • Charge serveur élevée
  • Problème réseau

Solutions :

  • Réduisez la taille du fichier ou divisez-le
  • Augmentez le timeout de votre client HTTP
  • Réessayez plus tard

Rate Limiting

Limite de taux dépassée

Code HTTP : 429

{
  "detail": "Trop de requêtes. Veuillez patienter avant de réessayer."
}

Cause : Vous avez dépassé le nombre de requêtes autorisées dans un laps de temps donné.

Solutions :

  • Attendez quelques secondes/minutes avant de réessayer
  • Implémentez un système de retry avec backoff exponentiel
  • Espacez vos requêtes au lieu de les envoyer toutes simultanément
  • Utilisez le batch processing pour optimiser le nombre de requêtes
  • Contactez le support pour augmenter votre quota si nécessaire

Exemple de retry avec backoff exponentiel (Python)

import time
import requests

def call_api_with_retry(url, headers, data, max_retries=3):
    for attempt in range(max_retries):
        try:
            response = requests.post(url, headers=headers, json=data)
            if response.status_code == 429:
                wait_time = 2 ** attempt  # 1, 2, 4 secondes
                print(f"Rate limit, attente de {wait_time}s...")
                time.sleep(wait_time)
                continue
            response.raise_for_status()
            return response.json()
        except requests.exceptions.HTTPError as e:
            if attempt == max_retries - 1:
                raise
    raise Exception("Max retries atteint")

Guide de dépannage

Comment diagnostiquer une erreur ?

  1. Notez le code HTTP : Indique le type d'erreur général
  2. Lisez le message "detail" : Donne la cause spécifique
  3. Vérifiez vos paramètres : Assurez-vous que tous les champs requis sont présents
  4. Testez avec curl : Isolez le problème en testant avec une requête simple
  5. Consultez les logs : Vérifiez vos logs applicatifs pour plus de contexte

Erreurs persistantes ?

Si une erreur persiste malgré vos corrections :

  • Vérifiez le statut de l'API (page de statut si disponible)
  • Testez votre clé API avec un endpoint simple (GET /languages/)
  • Vérifiez que votre abonnement est actif
  • Contactez le support avec les informations suivantes :
    • Code HTTP de l'erreur
    • Message "detail" complet
    • Endpoint appelé
    • Exemple de requête (sans votre clé API)
    • Timestamp de l'erreur

Bonnes pratiques de gestion d'erreurs

Implémentez une gestion robuste

// Exemple JavaScript
async function translateWithErrorHandling(text, sourceLang, targetLang) {
    try {
        const response = await fetch('https://portail.lexamt.fr/api/v1/translate/', {
            method: 'POST',
            headers: {
                'Authorization': `Bearer ${API_KEY}`,
                'Content-Type': 'application/json'
            },
            body: JSON.stringify({
                action: 'text_translate',
                source_language: sourceLang,
                target_language: targetLang,
                text: text
            })
        });

        if (!response.ok) {
            const error = await response.json();

            // Gestion par code HTTP
            switch (response.status) {
                case 400:
                    console.error('Erreur de validation:', error.detail);
                    // Afficher un message convivial à l'utilisateur
                    break;
                case 401:
                    console.error('Erreur d\'authentification:', error.detail);
                    // Rediriger vers la page de configuration
                    break;
                case 429:
                    console.error('Rate limit:', error.detail);
                    // Réessayer après un délai
                    break;
                case 500:
                    console.error('Erreur serveur:', error.detail);
                    // Réessayer automatiquement
                    break;
                default:
                    console.error('Erreur inconnue:', error.detail);
            }

            throw new Error(error.detail);
        }

        return await response.json();
    } catch (error) {
        console.error('Erreur réseau ou parsing:', error);
        throw error;
    }
}

Loguez les erreurs

  • Enregistrez le code HTTP, le message d'erreur et le contexte
  • N'enregistrez JAMAIS la clé API complète dans les logs
  • Incluez un timestamp pour faciliter le débogage
  • Utilisez des niveaux de log appropriés (ERROR, WARNING, etc.)

Affichez des messages conviviaux

Ne montrez pas les messages techniques bruts aux utilisateurs finaux. Transformez-les en messages compréhensibles :

  • 401/Clé invalide → "Votre clé API est invalide. Veuillez vérifier vos paramètres."
  • 400/Texte trop long → "Le texte à traduire est trop long (max 2000 caractères)."
  • 429/Rate limit → "Trop de traductions en cours. Veuillez patienter quelques instants."
  • 500/Erreur serveur → "Service temporairement indisponible. Veuillez réessayer."
