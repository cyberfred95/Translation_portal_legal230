Endpoint de traduction

POST /api/v1/translate/

Cet endpoint polyvalent permet de traduire du texte ou des fichiers.

URL complète

https://portail.lexamt.fr/api/v1/translate/

Headers requis

Authorization: Bearer VOTRE_CLE_API
Content-Type: application/json

Traduction de texte

Body (JSON) :

{
  "action": "text_translate",
  "source_language": "en",
  "target_language": "fr",
  "text": "Texte à traduire",
  "domain_name": "Contract Law" (optionnel)
}

Réponse (200 OK) :

{
  "translated_text": "Texte traduit"
}

Traduction de fichiers

Body (JSON avec fichiers en base64) :

{
  "action": "file_translate",
  "source_language": "en",
  "target_language": "fr",
  "document": ["base64_encoded_file"],
  "domain_name": "Contract Law" (optionnel)
}

Réponse (200 OK) :

[
  {
    "project_id": "12345",
    "status": "Translated",
    "translated_file_url": "https://portail.lexamt.fr/media/..."
  }
]

Paramètres

Paramètre Type Requis Description
action string Oui "text_translate" ou "file_translate"
source_language string Oui Code ISO de la langue source (max 10 char)
target_language string Oui Code ISO de la langue cible (max 10 char)
text string Si text_translate Texte à traduire (max 2000 char)
document array Si file_translate Fichiers encodés en base64 (max 10 fichiers)
domain_name string Non Nom du domaine de spécialisation (max 128 char)

Codes de réponse

  • 200 OK : Traduction réussie
  • 400 Bad Request : Erreur de validation (champ manquant, format invalide, etc.)
  • 401 Unauthorized : Authentification échouée

GET /api/v1/domain/{id}/glossaries/

Retourne les glossaires par défaut pour un domaine spécifique.

URL complète

https://portail.lexamt.fr/api/v1/domain/{id_domain}/glossaries/

Paramètres d'URL

  • id_domain : ID du domaine (entier)

Paramètres de requête (optionnels)

  • source_language : Filtrer par langue source
  • target_language : Filtrer par langue cible

Exemple

GET /api/v1/domain/1/glossaries/?source_language=en&target_language=fr

Réponse (200 OK)

[
  {
    "id": 10,
    "file_url": "https://portail.lexamt.fr/media/glossaries/...",
    "name": "Glossaire Droit des Contrats EN-FR",
    "source_language": "en",
    "target_language": "fr",
    "file_size": "1.2 MB",
    "created_at": "2024-01-15T10:30:00Z"
  }
]

POST /api/v1/glossary/

Crée un nouveau glossaire personnel.

URL complète

https://portail.lexamt.fr/api/v1/glossary/

Headers requis

Authorization: Bearer VOTRE_CLE_API
Content-Type: application/json

Body (JSON)

{
  "name": "Mon glossaire",
  "source_language": "en",
  "target_language": "fr",
  "file": "base64_encoded_csv_or_xlsx"
}

Formats acceptés

  • CSV (text/csv)
  • XLSX (application/vnd.openxmlformats-officedocument.spreadsheetml.sheet)

Limites

  • Taille maximale : 10MB
  • Nom : maximum 128 caractères

Réponse (200 OK)

{
  "id": 43,
  "name": "Mon glossaire",
  "source_language": "en",
  "target_language": "fr",
  "created_at": "2024-10-26T15:00:00Z"
}

GET /api/v1/glossary/{id}/

Récupère les détails d'un glossaire spécifique.

URL complète

https://portail.lexamt.fr/api/v1/glossary/{id_glossary}/

Paramètres d'URL

  • id_glossary : ID du glossaire (entier)

Codes de réponse

  • 200 OK : Glossaire trouvé
  • 404 Not Found : Glossaire inexistant ou non accessible

DELETE /api/v1/glossary/{id}/

Supprime un glossaire.

URL complète

https://portail.lexamt.fr/api/v1/glossary/{id_glossary}/

Codes de réponse

  • 204 No Content : Glossaire supprimé avec succès
  • 404 Not Found : Glossaire inexistant

Schémas de données

Pour les schémas complets de tous les objets de données (structures JSON, validations, types, etc.), consultez la section components/schemas du fichier Swagger.

Principaux schémas

  • TranslateTextRequest : Requête de traduction de texte
  • TranslateFileRequest : Requête de traduction de fichiers
  • TranslateTextResponse : Réponse de traduction de texte
  • TranslateFileResponse : Réponse de traduction de fichiers
  • GlossaryRequest : Requête de création de glossaire
  • GlossaryResponse : Réponse de glossaire
  • DefaultGlossary : Glossaire par défaut
  • Domain : Domaine de spécialisation
  • Error : Format d'erreur standard

Voir les schémas complets dans Swagger

Règles de validation

Limites de longueur

Champ Limite
action 32 caractères
text 2000 caractères
source_language / target_language 10 caractères
domain_name 128 caractères
name (glossaire) 128 caractères

Limites de fichiers

Type Limite
Fichier de traduction 50MB maximum
Nombre de fichiers par requête 10 maximum
Fichier glossaire 10MB maximum

Valeurs enum

  • action : "text_translate" | "file_translate"

Utiliser le fichier Swagger

Avec Swagger Editor

  1. Accédez à editor.swagger.io
  2. Cliquez sur File > Import File
  3. Sélectionnez le fichier swagger.yaml
  4. Explorez la documentation interactive

Avec Swagger UI

Pour héberger votre propre Swagger UI :

docker run -p 80:8080 -e SWAGGER_JSON=/swagger.yaml \
  -v /path/to/swagger.yaml:/swagger.yaml \
  swaggerapi/swagger-ui

Génération de code client

Utilisez Swagger Codegen pour générer automatiquement des clients API dans votre langage :

swagger-codegen generate -i swagger.yaml -l python -o ./client

Langages supportés : Python, JavaScript, Java, PHP, Ruby, Go, C#, etc.
