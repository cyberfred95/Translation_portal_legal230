Guide de démarrage rapide

LEXA API - Démarrez en 10 minutes

← Retour à la documentation

Introduction

Ce guide vous permettra d'effectuer votre première requête à LEXA API en moins de 10 minutes. Vous apprendrez à :

  • Obtenir votre clé API
  • Faire votre première requête de traduction
  • Gérer les langues et domaines
  • Créer et utiliser des glossaires

Prérequis

  • Un compte LEXA actif sur portail.lexamt.fr
  • Un abonnement LEXA API (voir Forfait LEXA API)
  • Un outil pour tester les API (curl, Postman, ou votre langage de programmation préféré)
  • Une connexion Internet

Étape 1 : Obtenir votre clé API

Créer un compte et souscrire

  1. Rendez-vous sur portail.lexamt.fr
  2. Créez un compte si ce n'est pas déjà fait
  3. Souscrivez à un abonnement LEXA API
  4. Connectez-vous à votre compte

Récupérer votre clé API

  1. Une fois connecté, cliquez sur Paramètres dans le menu
  2. Accédez à la section API ou Clé API
  3. Votre clé API s'affiche (ou cliquez sur Générer une clé si nécessaire)
  4. Cliquez sur Copier pour copier la clé dans votre presse-papier
  5. Conservez cette clé en sécurité (ne jamais la partager publiquement)

Exemple de clé API : abc123def456ghi789jkl012mno345pq

Note : Votre clé API est unique et confidentielle. Ne la partagez jamais dans du code public, sur GitHub, ou dans des forums.

Étape 2 : Tester l'authentification

Vérifier que votre clé fonctionne

Testons votre clé API en récupérant la liste des langues disponibles :

Avec curl :

curl -X GET https://portail.lexamt.fr/api/v1/languages/ \
  -H "Authorization: Bearer VOTRE_CLE_API"

Réponse attendue :

[
  {
    "name": "English",
    "language_code": "en"
  },
  {
    "name": "French",
    "language_code": "fr"
  },
  {
    "name": "Spanish",
    "language_code": "es"
  }
  ...
]

✓ Si vous obtenez cette réponse : Votre clé API fonctionne correctement !

✗ Si vous obtenez une erreur : Vérifiez que vous avez bien copié la clé complète et que le header Authorization est correct.

Étape 3 : Première traduction de texte

Traduire un texte simple

Traduisons un texte de l'anglais vers le français :

Avec curl :

curl -X POST https://portail.lexamt.fr/api/v1/translate/ \
  -H "Authorization: Bearer VOTRE_CLE_API" \
  -H "Content-Type: application/json" \
  -d '{
    "action": "text_translate",
    "source_language": "en",
    "target_language": "fr",
    "text": "This is a legal contract between the parties."
  }'

Réponse attendue :

{
  "translated_text": "Il s'agit d'un contrat juridique entre les parties."
}

Paramètres de la requête

  • action : "text_translate" pour traduire du texte
  • source_language : Code de la langue source (ex: "en")
  • target_language : Code de la langue cible (ex: "fr")
  • text : Le texte à traduire (max 2000 caractères)
  • domain_name : (Optionnel) Domaine de spécialisation

Étape 4 : Traduction avec domaine spécialisé

Récupérer les domaines disponibles

Listons d'abord les domaines juridiques disponibles :

Avec curl :

curl -X GET https://portail.lexamt.fr/api/v1/domains/ \
  -H "Authorization: Bearer VOTRE_CLE_API"

Réponse attendue :

[
  {
    "id": 1,
    "domain_group": "Legal",
    "name": "Contract Law"
  },
  {
    "id": 2,
    "domain_group": "Legal",
    "name": "Intellectual Property"
  }
  ...
]

Traduire avec un domaine

Utilisons maintenant un domaine pour améliorer la qualité de la traduction :

Avec curl :

curl -X POST https://portail.lexamt.fr/api/v1/translate/ \
  -H "Authorization: Bearer VOTRE_CLE_API" \
  -H "Content-Type: application/json" \
  -d '{
    "action": "text_translate",
    "source_language": "en",
    "target_language": "fr",
    "text": "The party of the first part agrees to indemnify the party of the second part.",
    "domain_name": "Contract Law"
  }'

Avantage : L'utilisation d'un domaine adapte la terminologie juridique pour une traduction plus précise.

Étape 5 : Traduction de fichiers

Préparer votre fichier

Pour traduire un fichier, vous devez l'encoder en base64. Voici comment faire :

Sur Linux/Mac :

base64 -i document.docx -o document_base64.txt

Sur Windows (PowerShell) :

[Convert]::ToBase64String([IO.File]::ReadAllBytes("document.docx")) > document_base64.txt

Envoyer le fichier pour traduction

Avec curl :

curl -X POST https://portail.lexamt.fr/api/v1/translate/ \
  -H "Authorization: Bearer VOTRE_CLE_API" \
  -H "Content-Type: application/json" \
  -d '{
    "action": "file_translate",
    "source_language": "en",
    "target_language": "fr",
    "document": ["'$(cat document_base64.txt)'"]
  }'

Réponse attendue :

[
  {
    "project_id": "12345",
    "status": "Translated",
    "translated_file_url": "https://portail.lexamt.fr/media/translations/document_fr.docx"
  }
]

Formats supportés

  • TXT (texte brut)
  • DOCX (Microsoft Word)
  • XLSX (Microsoft Excel)
  • PPTX (Microsoft PowerPoint)
  • PDF (Adobe PDF)

Limites

  • Taille maximale : 50MB par fichier
  • Maximum 10 fichiers par requête

Étape 6 : Créer un glossaire personnalisé

Préparer votre fichier glossaire

Créez un fichier CSV ou Excel avec deux colonnes : terme source et terme cible.

Exemple CSV (glossaire.csv) :

contract,contrat
party,partie
indemnify,indemniser
liability,responsabilité

Encoder le fichier en base64

base64 -i glossaire.csv -o glossaire_base64.txt

Créer le glossaire via l'API

Avec curl :

curl -X POST https://portail.lexamt.fr/api/v1/glossary/ \
  -H "Authorization: Bearer VOTRE_CLE_API" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Mon glossaire juridique EN-FR",
    "source_language": "en",
    "target_language": "fr",
    "file": "'$(cat glossaire_base64.txt)'"
  }'

Réponse attendue :

{
  "id": 42,
  "name": "Mon glossaire juridique EN-FR",
  "source_language": "en",
  "target_language": "fr",
  "created_at": "2024-10-26T14:30:00Z"
}

Lister vos glossaires

curl -X GET https://portail.lexamt.fr/api/v1/glossaries/ \
  -H "Authorization: Bearer VOTRE_CLE_API"

Exemples complets par langage

Python

import requests
import json

API_KEY = "votre_cle_api"
BASE_URL = "https://portail.lexamt.fr/api/v1"

headers = {
    "Authorization": f"Bearer {API_KEY}",
    "Content-Type": "application/json"
}

# Traduction de texte
data = {
    "action": "text_translate",
    "source_language": "en",
    "target_language": "fr",
    "text": "This is a legal contract.",
    "domain_name": "Contract Law"
}

response = requests.post(f"{BASE_URL}/translate/", headers=headers, json=data)
result = response.json()
print(result["translated_text"])

JavaScript (Node.js)

const axios = require('axios');

const API_KEY = 'votre_cle_api';
const BASE_URL = 'https://portail.lexamt.fr/api/v1';

const headers = {
    'Authorization': `Bearer ${API_KEY}`,
    'Content-Type': 'application/json'
};

// Traduction de texte
const data = {
    action: 'text_translate',
    source_language: 'en',
    target_language: 'fr',
    text: 'This is a legal contract.',
    domain_name: 'Contract Law'
};

axios.post(`${BASE_URL}/translate/`, data, { headers })
    .then(response => {
        console.log(response.data.translated_text);
    })
    .catch(error => {
        console.error('Error:', error.response.data);
    });

PHP

<?php
$apiKey = 'votre_cle_api';
$baseUrl = 'https://portail.lexamt.fr/api/v1';

$data = array(
    'action' => 'text_translate',
    'source_language' => 'en',
    'target_language' => 'fr',
    'text' => 'This is a legal contract.',
    'domain_name' => 'Contract Law'
);

$options = array(
    'http' => array(
        'header'  => "Authorization: Bearer $apiKey\r\n" .
                     "Content-Type: application/json\r\n",
        'method'  => 'POST',
        'content' => json_encode($data)
    )
);

$context  = stream_context_create($options);
$result = file_get_contents("$baseUrl/translate/", false, $context);
$response = json_decode($result, true);

echo $response['translated_text'];
?>

Résolution de problèmes

Erreur d'authentification

Symptôme : {"detail": "Header Authorization requis"}

Solution :

  • Vérifiez que le header Authorization est présent
  • Format correct : Authorization: Bearer VOTRE_CLE
  • Pas d'espace supplémentaire avant ou après la clé

Clé API invalide

Symptôme : {"detail": "Clé API invalide"}

Solution :

  • Vérifiez que vous avez copié la clé complète
  • Assurez-vous que votre abonnement est actif
  • Régénérez une nouvelle clé si nécessaire

Champ requis manquant

Symptôme : {"detail": "Le champ 'text' est requis..."}

Solution :

  • Vérifiez que tous les champs requis sont présents
  • Pour text_translate : action, source_language, target_language, text
  • Pour file_translate : action, source_language, target_language, document

Fichier trop volumineux

Symptôme : Erreur de taille de fichier

Solution :

  • Vérifiez que le fichier fait moins de 50MB
  • Compressez le fichier si possible
  • Divisez en plusieurs fichiers plus petits

Pour plus d'aide, consultez Codes d'erreur ou contactez support@lexamt.fr.
